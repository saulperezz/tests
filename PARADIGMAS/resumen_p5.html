Práctica 5: Funciones de Orden Superior

LPP · Práctica 5·5 conceptos clave
1
map, filter, foldr, foldl
Concepto
Funciones de orden superior que aplican una operación a cada elemento de una lista (map), filtran elementos (filter), o reducen la lista a un valor (foldr/foldl).
(map (lambda (x) (* x 2)) '(1 2 3)) ; => (2 4 6) (filter (lambda (x) (> x 3)) '(1 5 2 8)) ; => (5 8) (foldr + 0 '(1 2 3 4)) ; => 10
2
aplica-veces y composición
Concepto
Componer funciones para aplicarlas repetidamente o en secuencia.
(aplica-veces doble suma-2 3 5) ; => 68
3
Ordenación genérica
Concepto
Crear funciones de orden superior que ordenan listas usando diferentes comparadores.
(ordena-generica < '(3 1 4 1 5)) ; => (1 1 3 4 5)
4
Closures
Concepto
Una función que "captura" variables de su entorno externo. Las lambdas anidadas son closures.
(define f (lambda (x) (lambda (y) (+ x y)))) ((f 3) 5) ; => 8 (la función interna captura x=3)
5
FOS rewrites y apply
Concepto
Reescribir funciones recursivas usando funciones de orden superior (map, filter, foldr). apply aplica una función a una lista de argumentos.
(apply + '(1 2 3 4)) ; => 10
