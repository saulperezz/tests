Ejercicio 1 — Conversión binario-decimal y binario-hexadecimal
›
a) binario-a-decimal: 4 bits → número decimal
Convierte un número binario de 4 bits a su representación decimal. Cada bit se multiplica por su potencia de 2 correspondiente (2³, 2², 2¹, 2⁰).
(define (binario-a-decimal b3 b2 b1 b0) (+ (* b3 (expt 2 3)) (* b2 (expt 2 2)) (* b1 (expt 2 1)) (* b0 (expt 2 0))))
b) binario-a-hexadecimal: 4 bits → carácter hexadecimal
Convierte un número binario a hexadecimal. La estrategia es:
  • Convertir primero a decimal con binario-a-decimal
  • Si decimal ≤ 9: el carácter es (integer->char (+ decimal (char->integer #\0)))
  • Si decimal ≥ 10: el carácter es (integer->char (+ (- decimal 10) (char->integer #\A)))
(define (decimal-a-hexadecimal n) (if (<= n 9) (integer->char (+ n (char->integer #\0))) (integer->char (+ (- n 10) (char->integer #\A))))) (define (binario-a-hexadecimal b3 b2 b1 b0) (decimal-a-hexadecimal (binario-a-decimal b3 b2 b1 b0)))
Ejercicio 2 — Cifrado César
›
Implementación completa del cifrado César. Consiste en funciones auxiliares que trabajan juntas para cifrar y descifrar caracteres.
;;; Devuelve el índice 0..25 de un carácter minúscula (define (encuentra-indice char) (- (char->integer char) (char->integer #\a))) ;;; Devuelve el carácter minúscula correspondiente al índice 0..25 (define (encuentra-caracter indice) (integer->char (+ indice (char->integer #\a)))) ;;; Comprueba si un carácter está entre #\a y #\z (define (entre-az? char) (and (char>=? char #\a) (char<=? char #\z))) ;;; Rota un índice 0..25 aplicando el desplazamiento (puede ser negativo) (define (rota-indice indice desplazamiento) (modulo (+ indice desplazamiento) 26)) ;;; Cifra un carácter minúscula con el desplazamiento dado (define (cifra-caracter-minuscula char desplazamiento) (encuentra-caracter (rota-indice (encuentra-indice char) desplazamiento)))
Cifra un carácter:
  • Si es minúscula: cifra directamente
  • Si es mayúscula: baja a minúscula, cifra y sube a mayúscula
  • Si no es letra inglesa: devuelve el carácter sin cambios

Ejercicio 3 — Menor de tres
›
Dos formas de implementar la función que devuelve el menor de tres números: usando if anidado, o componiendo una función auxiliar.
Versión 1: usando if
(define (menor-de-tres n1 n2 n3) (if (< n1 n2) (if (< n1 n3) n1 n3) (if (< n2 n3) n2 n3))) (menor-de-tres 2 8 1) => 1 (menor-de-tres 3 0 3) => 0 (menor-de-tres 5 5 5) => 5 (menor-de-tres 10 1 100) => 1
Versión 2: composición de llamadas a menor
;;; Función auxiliar para la versión 2 (define (menor x y) (if (< x y) x y)) ;;; Versión 2: sin if en la función principal (define (menor-de-tres-v2 n1 n2 n3) (menor n1 (menor n2 n3))) (menor-de-tres-v2 2 8 1) => 1 (menor-de-tres-v2 3 0 3) => 0 (menor-de-tres-v2 5 5 5) => 5 (menor-de-tres-v2 10 1 100) => 1
Ejercicio 4 — Modelo de sustitución
›
a) Evaluación de (g (f (+ 2 1)) (+ 1 1))
Comparación entre orden aplicativo (evalúa argumentos antes de sustituir) y orden normal (sustituye primero, evalúa al final).

Considerando: (define (f x) (cons x 2)) y (define (g x y) (cons x y))
---- ORDEN APLICATIVO (evalúa argumentos antes de sustituir) ---- (g (f (+ 2 1)) (+ 1 1)) => (g (f 3) (+ 1 1)) ; se evalúa (+ 2 1) => 3 => (g (f 3) 2) ; se evalúa (+ 1 1) => 2 => (g (cons 3 2) 2) ; se evalúa (f 3) => (cons 3 2) => (cons (cons 3 2) 2) ; se sustituye en g => ((3 . 2) . 2) ; resultado final ---- ORDEN NORMAL (sustituye primero, evalúa al final) ---- (g (f (+ 2 1)) (+ 1 1)) => (cons (f (+ 2 1)) (+ 1 1)) ; se expande g => (cons (cons (+ 2 1) 2) (+ 1 1)) ; se expande f => (cons (cons 3 2) (+ 1 1)) ; se evalúa (+ 2 1) => 3 => (cons (cons 3 2) 2) ; se evalúa (+ 1 1) => 2 => ((3 . 2) . 2) ; resultado final Ambos órdenes producen el mismo resultado: '((3 . 2) . 2)
b) Evaluación de (func-2 0 (func-1 10))
Diferencia crítica entre órdenes de evaluación cuando hay efectos secundarios (errores).

Considerando: (define (func-1 x) (/ x 0)) y (define (func-2 x y) (if (= x 0) 0 y))
---- ORDEN APLICATIVO ---- (func-2 0 (func-1 10)) => (func-2 0 (/ 10 0)) ; se evalúa (func-1 10) => ERROR: división por cero ; (/ 10 0) provoca un error El programa termina con error antes de poder evaluar func-2. ---- ORDEN NORMAL ---- (func-2 0 (func-1 10)) => (if (= 0 0) 0 (func-1 10)) ; se expande func-2 => (if #t 0 (func-1 10)) ; se evalúa (= 0 0) => #t => 0 ; la rama #t devuelve 0 (func-1 10) NUNCA se evalúa porque if cortocircuita. Resultado final: 0 Conclusión: con orden normal el programa devuelve 0 sin error, con orden aplicativo produce un error de división por cero.
