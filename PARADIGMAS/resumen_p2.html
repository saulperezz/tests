Práctica 2: Programación Funcional

LPP · Práctica 2 · 5 conceptos clave
1
Conversión Binario-Decimal
Concepto
Un número binario de 4 bits se convierte a decimal multiplicando cada bit por su potencia de 2 correspondiente y sumando: b3×2³ + b2×2² + b1×2¹ + b0×2⁰.
Explicación
Cada posición en un número binario representa una potencia de 2. Por ejemplo:
  • 1111 = 1×8 + 1×4 + 1×2 + 1×1 = 15
  • 0110 = 0×8 + 1×4 + 1×2 + 0×1 = 6
  • 1000 = 1×8 + 0×4 + 0×2 + 0×1 = 8
Expresiones clave
(binario-a-decimal 1 1 1 1) ; => 15 (binario-a-decimal 0 1 1 0) ; => 6 (binario-a-decimal 1 0 0 0) ; => 8
2
Conversión Binario-Hexadecimal
Concepto
Un número binario se convierte a hexadecimal: primero se convierte a decimal, y luego a carácter hex: 0-9 y A-F.
Explicación
La estrategia es composición de funciones: primero convertimos binario a decimal, luego decimal a hexadecimal:
  • Si el decimal es 0-9, el carácter es el dígito (0, 1, ... 9)
  • Si el decimal es 10-15, el carácter es la letra (A, B, C, D, E, F)
Expresiones clave
(binario-a-hexadecimal 1 1 1 1) ; => #\F (binario-a-hexadecimal 0 1 1 0) ; => #\6 (binario-a-hexadecimal 1 0 1 0) ; => #\A
3
Cifrado César
Concepto
El cifrado César desplaza cada letra un número fijo de posiciones en el alfabeto. Un desplazamiento de 3 convierte A→D, B→E, etc.
Explicación
La implementación maneja:
  • Letras minúsculas directamente: desplaza dentro del rango a-z con módulo 26
  • Letras mayúsculas: convierte a minúscula, cifra, y vuelve a mayúscula
  • No-letras: devuelve sin cambios (números, signos, caracteres especiales)
El desplazamiento puede ser negativo (descifrado).
Expresiones clave
(cifra-caracter #\c 5) ; => #\h (cifra-caracter #\z -1) ; => #\y (descifra-caracter #\d 3) ; => #\a (cifra-caracter #\D 3) ; => #\G
4
Función Menor de Tres
Concepto
Encontrar el menor de tres números puede hacerse con if anidado, o componiendo una función auxiliar menor.
Explicación
Hay dos enfoques:
  • Versión 1: usar if anidado: comparar el primero con el segundo, luego el resultado con el tercero
  • Versión 2: definir una función menor que compara dos números, luego componer: (menor n1 (menor n2 n3))
Ambas producen el mismo resultado, pero la versión 2 es más modular y reutilizable.
Expresiones clave
(menor-de-tres 2 8 1) ; => 1 (menor-de-tres 3 0 3) ; => 0 (menor-de-tres 5 5 5) ; => 5 (menor-de-tres 10 1 100) ; => 1
5
Clasificación de Manos de Cartas
Concepto
Una función que clasifica una mano de 3 cartas: trío (todas iguales), pareja (dos iguales), o nada.
Explicación
La función extrae el valor numérico de cada carta de baraja usando valor-carta, y luego:
  • Si los tres valores son iguales → trío
  • Si dos valores son iguales → pareja
  • Si todos diferentes → nada
Los ases valen 1, figuras valen 10-12, y número vales su valor (2-9). Los palos no importan.
Expresiones clave
(jugada-mano '3♥ '3♣ '3♥) ; => "trío de 3" (jugada-mano 'K♦ '7♠ 'K♥) ; => "pareja de 12" (jugada-mano '5♣ '4♣ '6♣) ; => "nada"
