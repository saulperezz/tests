Divide y Vencerás

ADA · Tema 3 · 14 conceptos clave
1
Paradigma Divide y Vencerás
Concepto
Técnica de diseño que resuelve un problema descomponiéndolo en subproblemas de menor tamaño, resolviéndolos de forma independiente y combinando sus soluciones para obtener la solución al problema original.
Explicación
Las tres fases son siempre:
  1. Divide: partir el problema de talla n en h subproblemas de talla n/b.
  2. Conquista (resuelve): resolver cada subproblema recursivamente. Si el problema es suficientemente pequeño (caso base), resolverlo directamente de forma trivial.
  3. Combina: fusionar las soluciones de los subproblemas para producir la solución global.
Para que sea aplicable deben existir: una forma de descomponer el problema, una solución trivial para los casos base y una forma de combinar soluciones. No siempre un subproblema de talla menor es más fácil, ni siempre combinar soluciones es sencillo — estas son las dos dificultades del paradigma.
2
Esquema General de Divide y Vencerás
Concepto
Plantilla algorítmica que describe la estructura común a cualquier algoritmo Divide y Vencerás, independientemente del problema concreto.
Esquema
Solution DC(Problem p) { if (is_simple(p)) // caso base return trivial(p); list<Solution> s; for (Problem q : divide(p)) // dividir s.push_back(DC(q)); // conquistar (recursión) return combine(s); // combinar }
Explicación
Para instanciar este esquema hay que definir cuatro elementos:
  • is_simple(p): condición de parada (caso trivial).
  • trivial(p): solución directa del caso base.
  • divide(p): cómo partir el problema en subproblemas.
  • combine(s): cómo fusionar las soluciones parciales.
Ejemplo Mergesort: is_simple = tamaño ≤ 1; trivial = devolver sin hacer nada; divide = partir por la mitad; combine = función merge.
3
Ecuación de Recurrencia General de DyV
Concepto
Relación matemática que expresa la complejidad de un algoritmo DyV en función del número de subproblemas, su tamaño y el coste de dividir y combinar.
Forma general
T(n) = h · T(n/b) + g(n)
Explicación
  • h = número de subproblemas en que se divide el problema.
  • b = factor de reducción del tamaño (cada subproblema tiene talla n/b).
  • g(n) = coste de dividir y combinar, sin contar las llamadas recursivas.
La complejidad final depende del equilibrio entre cuánto trabajo extra genera cada nivel (g(n)) y cuántos subproblemas se producen por nivel (h).
4
Teorema Maestro (Master Theorem)
Concepto
Fórmula cerrada que resuelve directamente T(n) = h·T(n/b) + g(n) cuando g(n) ∈ Θ(nᵏ), comparando el crecimiento de subproblemas con el del trabajo de combinar.
Resultado
CondiciónT(n)Intuición
h < bᵏΘ(nᵏ)Domina el trabajo de combinar (nivel raíz)
h = bᵏΘ(nᵏ · log_b n)Trabajo equilibrado en todos los niveles
h > bᵏΘ(n^(log_b h))Domina el número de subproblemas (hojas)

El teorema compara la velocidad de crecimiento del número de subproblemas (hⁱ a nivel i) con la velocidad de decrecimiento del trabajo por subproblema (g(n/bⁱ)). Si h = bᵏ, todos los niveles contribuyen igual y aparece el factor logarítmico.
5
Teorema de Reducción (Caso b = h = a)
Concepto
Simplificación del Teorema Maestro para el caso más favorable: el problema se divide en a subproblemas del mismo tamaño (n/a cada uno), con g(n) ∈ Θ(nᵏ).
Resultado
CondiciónT(n)Ejemplo práctico
k > 1Θ(nᵏ)Domina combinar (coste cuadrático o superior)
k = 1Θ(n log n)Mergesort: combinar lineal, caso óptimo habitual
k < 1Θ(n)Combinar sublineal: la recursión sigue siendo lineal

El resultado más sorprendente es k < 1: aunque haya múltiples llamadas recursivas, si el coste de combinar es sublineal, la complejidad total sigue siendo Θ(n).
6
Mergesort
Concepto
Algoritmo de ordenación DyV que divide el array por la mitad, ordena cada mitad recursivamente y fusiona (merge) las dos mitades ordenadas. Garantiza Θ(n log n) en todos los casos.
Explicación
  • Divide: partir el array en dos mitades de igual tamaño (índice medio).
  • Conquista: llamadas recursivas a mergesort sobre cada mitad.
  • Combina: función merge que fusiona dos subarrays ordenados en O(n).
Recurrencia: T(n) = 2T(n/2) + Θ(n) → con a=2, k=1 en el teorema de reducción: T(n) ∈ Θ(n log n).

Complejidad espacial: Θ(n) — la función merge necesita un array auxiliar. Los bloques de distintos niveles nunca coexisten; el mayor es el del nivel raíz. Más O(log n) de pila de recursión.

Diferencia clave con Quicksort: siempre divide en mitades iguales → garantiza Θ(n log n) en todos los casos, a costa de O(n) de memoria extra.
7
Función Merge
Concepto
Subrutina clave de Mergesort que fusiona dos subarrays ya ordenados en un único subarray ordenado usando un array auxiliar temporal.
Explicación
El algoritmo recorre simultáneamente los dos subarrays con dos punteros (uno por mitad), copiando en cada paso el elemento menor al array auxiliar, hasta agotar una de las mitades; luego copia el resto de la mitad no agotada.

Complejidad temporal: Θ(n) — cada elemento se copia exactamente una vez.
Complejidad espacial: Θ(n) — necesita un array auxiliar del tamaño total.

Es la operación de combinar del esquema DyV y la que determina el factor lineal g(n) = n en la recurrencia de Mergesort.
8
Quicksort como Divide y Vencerás
Concepto
Algoritmo de ordenación DyV donde toda la lógica está en la fase de dividir (particionar alrededor del pivote) y no se necesita fase de combinar.
Explicación
Instanciación del esquema DyV:
  • is_simple: tamaño ≤ 1.
  • divide: calcular la posición del pivote. El pivote queda en su posición definitiva; los demás elementos quedan a su izquierda (menores) o derecha (mayores).
  • combine: no necesaria — el array ya queda ordenado in-place.
La división puede ser muy desigual:
  • Mejor caso (pivote = mediana) → subproblemas (n/2, n/2) → Ω(n log n).
  • Peor caso (pivote = mínimo o máximo) → subproblemas (0, n−1) → O(n²).
Sin fase de combinar, la complejidad espacial extra es solo la pila de recursión: O(log n) en el mejor caso y O(n) en el peor.
9
Reduce y Vencerás
Concepto
Variante de DyV en la que solo se resuelve uno de los subproblemas en cada nivel de recursión, descartando el otro porque se sabe que la solución no está ahí.
Explicación
Se aplica cuando la lógica del problema permite determinar en cuál de los subproblemas se encuentra la solución sin necesidad de explorar ambos. Como solo hay una llamada recursiva, no existe fase de combinar y la recurrencia toma la forma:
T(n) = T(n/b) + g(n)
El ejemplo paradigmático es la búsqueda binaria: en cada paso se calcula el elemento central, se compara con el buscado y se descarta la mitad irrelevante. Recurrencia: T(n) = T(n/2) + Θ(1) → T(n) ∈ O(log n).
10
Búsqueda Binaria (Búsqueda Dicotómica)
Concepto
Algoritmo de búsqueda sobre arrays ordenados que en cada paso descarta la mitad del array donde no puede estar el elemento buscado, logrando complejidad O(log n).
Explicación
  • Precondición: el array debe estar ordenado.
  • Divide: calcular el índice central (med = first + count/2).
  • Conquista: si v[med] < val → buscar en mitad derecha; si no → mitad izquierda. Solo se resuelve uno de los dos subproblemas.
  • Combina: no necesaria.
  • Caso base: count == 0 (intervalo vacío, elemento no encontrado).
Recurrencia: T(n) = T(n/2) + Θ(1) → T(n) ∈ O(log n).

Es óptima para arrays ordenados: ningún algoritmo de búsqueda por comparación puede hacerlo en menos de O(log n) en el peor caso.
11
Torres de Hanoi
Concepto
Problema clásico DyV donde la reducción es n−1 (no n/b), lo que da lugar a una complejidad exponencial inevitable.
Explicación
Mover n discos de la torre A a la C usando B como auxiliar equivale a:
  1. Mover n−1 discos de A a B.
  2. Mover el disco mayor de A a C.
  3. Mover n−1 discos de B a C.
Recurrencia: T(n) = 2T(n−1) + 1 → por sustitución: T(n) = 2ⁿ − 1 → T(n) ∈ Θ(2ⁿ).

No se puede aplicar el Teorema Maestro porque la reducción es n−1, no n/b. La complejidad exponencial es intrínseca al problema: cualquier algoritmo que lo resuelva necesita al menos 2ⁿ−1 movimientos.
12
Quickselect
Concepto
Algoritmo basado en el particionado de Quicksort para encontrar el k-ésimo elemento más pequeño de un array sin necesidad de ordenarlo completamente.
Explicación
Aplica el particionado de Quicksort: coloca el pivote en su posición definitiva p y compara con k:
  • Si k == p → el pivote es el k-ésimo elemento, devuelve v[k].
  • Si k < p → el elemento está en la parte izquierda → llamada recursiva solo sobre esa parte.
  • Si k > p → está en la parte derecha → llamada recursiva solo sobre esa parte.
Solo se explora uno de los dos subproblemas (Reduce y Vencerás). Complejidades:
  • Mejor caso: Ω(n) — el pivote cae exactamente en k.
  • Peor caso: O(n²) — el pivote siempre cae en un extremo.
  • Caso promedio (pivote en la mitad): Θ(n) — la más frecuente estadísticamente.
La STL de C++ incluye nth_element con comportamiento equivalente.
13
Exponenciación Rápida (Potencia n-ésima)
Concepto
Algoritmo DyV que calcula xⁿ en tiempo O(log n) aprovechando que xⁿ = (x^(n/2))² para n par, evitando las n−1 multiplicaciones del método directo.
Recurrencia
xⁿ = 1 si n = 0 xⁿ = x^(n/2) · x^(n/2) si n es par xⁿ = x^((n−1)/2) · x^((n−1)/2) · x si n es impar
Explicación
La clave está en reutilizar el resultado de x^(n/2) calculándolo una sola vez (almacenándolo en una variable). Si se hicieran dos llamadas recursivas separadas, no habría mejora.

Recurrencia: T(n) = T(n/2) + Θ(1) → T(n) ∈ O(log n).

Frente a la multiplicación directa (O(n) multiplicaciones), la mejora es enorme para exponentes grandes: calcular x¹⁰⁰⁰ requiere solo ~10 multiplicaciones.
